Техническое задание

План-капкан. Техническое задание позволяет:

  1. исполнителю:
    • суть
    • планирвоание
    • отказ не по ТЗ
  2. заказчику:
    • что нужно
    • соответствие продукта
  3. обоим сторонам:
    • представить готовый продукт;
    • проверка по ТЗ
    • нет ошибкам по измененным тебованиям

Пример прототипа

Главный игрок - проектировщики интерфейсов, UI/UX дизайнеры

UX - user experience

Дизайн пользовательского интерфейса = Визуальный дизайн + Дизайн взаимодействия.

  • UX делает интерфейсы полезными
  • UX помогает пользователям достигать целей

UX - user interface

UX - усовершенствование взаимодействий с помощью палитры красок

  • UI делает интерфейсы красивыми
  • UI создает эмоциональные связи.

Дазайн сайта

Дизайн сайта – это совокупность графических элементов, шрифтов и цветов, реализованных на сайте.

Ключевой игрок - дизайнер

  • Adobe Photoshop + Adobe
  • Sketch ( Mac OS ) (Corel DRAW, Adobe Illustrator)
  • ...

Frontend

Это специалист, который переводит всю красоту дизайнера в программный код для программиста.

HTML, CSS, JAVASCRIPT

Ключевой игрок - html-верстальщик,frontend-разработчик

Frontend

  • Немного разбираться в дизайне и мобильном дизайне
  • Понимать работу серверной части
  • Разбираться в работе сетей
  • Разбираться в производительности
  • Разбираться в тестировании
  • Разбираться в системах сборки
  • Разбираться в настройке веб-серверов

Backend

Бэкэнд development – это процесс программирования сайта и наполнения его функционалом.

Ключевой игрок - backend-программист. Разрабатывающий серверную логику и осознающий, в каком контексте она будет исполняться – в контексте веб-приложений.
